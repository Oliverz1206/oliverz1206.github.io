## 1. What is rbenv

`rbenv` 是一个 Ruby 版本管理工具，用来在同一台机器上管理多个 Ruby 版本，并且可以针对不同项目切换不同版本。

它本身做的事情很简单：

- 安装/管理多个 Ruby 版本
- 为当前 shell、当前目录、全局环境指定 Ruby 版本
- 让不同项目互不干扰

一个很常见的使用场景是：

- 项目 A 使用 Ruby 3.2.2
- 项目 B 使用 Ruby 3.4.8
- 系统自带 Ruby 继续保留，不直接修改

> `rbenv` 本质上不会“替换系统 Ruby”，而是通过 PATH 和 shim 机制，让你在不同目录下调用到不同版本的 Ruby。
{: .prompt-tip }

## 2. 常用命令总览

| Command               | Meaning                          |
| --------------------- | -------------------------------- |
| `rbenv versions`      | 查看当前已安装的 Ruby 版本       |
| `rbenv install 3.4.8` | 安装指定 Ruby 版本               |
| `rbenv global 3.4.8`  | 设置全局默认 Ruby 版本           |
| `rbenv local 3.4.8`   | 为当前项目设置本地 Ruby 版本     |
| `rbenv shell 3.4.8`   | 仅为当前终端临时切换版本         |
| `rbenv version`       | 查看当前实际生效的 Ruby 版本     |
| `rbenv which ruby`    | 查看当前 `ruby` 命令实际指向哪里 |
| `rbenv rehash`        | 重新生成 shim                    |

## 3. 安装与查看

安装一个 ruby 版本：

```bash
rbenv install 3.4.8
```

注：`rbenv` 安装的 Ruby 版本是集中放在 `~/.rbenv/versions/`。

检查 `rbenv` 当前管理了哪些版本：

```bash
$ rbenv versions
  system
* 3.4.8 (set by /Users/oliver/Website/.ruby-version)
```

这里的含义是：

- `system`：系统自带 Ruby，不是 `rbenv` 安装的版本。
- `3.4.8`：你安装的 Ruby 版本
- `*`：当前正在使用的版本
- `(set by /Users/oliver/Website/.ruby-version)`：说明这个版本是被当前项目目录里的 `.ruby-version` 文件指定的，也就是说，是这个文件选定了当前版本环境，**而不是当前版本被安装在哪个位置**。


## 4. 三种管理 ruby 版本的方式

### a. global：全局默认版本

```bash
rbenv global 3.4.8
```

适合：

- 没有进入特定项目时
- 希望终端默认使用某个 Ruby 版本

检查：

```bash
$ rbenv version
3.4.8 (set by /Users/yourname/.rbenv/version)
```

### b. local：项目级版本

在项目目录下执行：

```bash
cd ~/Website
rbenv local 3.4.8
```

这会在当前目录生成一个 `.ruby-version` 文件。文件内容通常就是：

```txt
3.4.8
```

以后你只要进入这个目录，Ruby 版本就会自动切到 3.4.8。

### c. shell：当前终端临时版本

```bash
rbenv shell 3.2.2
```

适合：

- 临时测试
- 不想修改 global 和 local 配置

关闭终端后一般就失效。

### d. `global`、`local`、`shell` 的优先级

优先级从高到低通常是: `shell > local > global > system`

例如：

- 你全局设置了 `3.4.8`
- 项目目录里 `.ruby-version` 写的是 `3.2.2`
- 当前 shell 又执行了 `rbenv shell 3.1.0`

那么最终生效的是 3.1.0 版本。

> 在当前目录下没有 `.ruby-version` 则会依次向父目录里寻找来指定版本。
{: .prompt-info }

### e. 确认 Ruby 版本的方法

#### i. 看版本

```bash
$ ruby -v
ruby 3.4.8 (2025-07-16 revision xxxx) +PRISM [arm64-darwin]
```

#### ii. 看 rbenv 判断结果

```bash
$ rbenv version
3.4.8 (set by /Users/oliver/Website/.ruby-version)
```

#### iii. 看实际执行路径

```bash
$ rbenv which ruby
/Users/oliver/.rbenv/versions/3.4.8/bin/ruby
```

这比单纯看 `ruby -v` 更直接，因为它会告诉你命令实际来自哪里。


## 5. 为项目配置一个 ruby 环境

如果还没有需要的ruby版本安装，先安装特定版本：

```bash
cd ~/Website
rbenv install 3.4.8
rbenv local 3.4.8
```

然后再安装 bundler：

```bash
gem install bundler
rbenv rehash
```

这里执行 `rbenv rehash` 的原因是：当你安装了新的 gem 命令后，`rbenv` 需要更新 shim，才能让这些命令被正确找到。

最后安装依赖：

```bash
# 这里推荐将 gem 依赖安装在项目本地，对文件进行物理隔离。
bundle config set path vendor/bundle
bundle install
```
